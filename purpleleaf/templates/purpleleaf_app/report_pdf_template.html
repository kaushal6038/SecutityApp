10.96.32.36 22

Description

                          A version of Dropbear SSH was identified to be out of date and potentially affected by known vulnerabilities. Although the vulnerabilities documented exist within special configurations and were not found to be exploitable, future changes to the system may cause these issues to become more severe.

Remediation

                      For servers where Dropbear is managed independently it is recommended that the service is updated to the latest version from the project website: https://matt.ucc.asn.au/dropbear/dropbear.html In many cases Dropbear is bundled with router firmware or other proprietary operating systems. In such cases it is recommended that the systems and packages be updated first to determine if a current patch level of Dropbear exists.

Low Risk

Default SNMP Public Community Name

Affected IPs:

IP ADDRESS PORT
10.96.32.237 161


Description

                          SNMP services were found to disclose internal system information. The Simple Network Management Protocol is often used to monitor devices across a network. SNMP versions 1, 2, and 2c do not offer security controls, allowing diagnostic information to be exposed to an attacker. Although it is uncommon for critical information to be disclosed within SNMP services, it provides information which is useful for many other attacks.

Remediation

                      Virtue Security suggests that the SNMP service is disabled if not required. In situations where SNMP is required the community string should be changed to non-dictionary value.

CONCLUSIONS

This vulnerability assessment is an analysis of the security posture as it was seen during the dates in which testing was conducted. Any future changes to the application or the infrastructure on which it resides may change the security posture from its current state. Additionally, as time passes, new types of attacks may arise which were previously unknown to the security industry. Because of this, Virtue Security recommends that all applications undergo recurring security testing within a reasonable timeframe.

Virtue Security also recommends that all issues identified in this report are remediated. Virtue Security makes a best effort to rank the severity of vulnerabilities; however, it is not possible for any assessment to guarantee that Low-risk issues cannot be used to significantly impact Acme Banking. All risk ratings in this report are a combination of industry standard ratings as well as considerations for unique business risk as identified by Virtue Security consultants. Special consideration should also be given to the fact that multiple low risk issues may compound into higher risk threats.

For most effective remediation, Virtue Security recommends developing processes or altering existing processes to include the fixes recommended in this report. Infrastructure related issues may be most effective when applied to baseline configurations which can be deployed across the organization. Application related fixes may be most effective with secure frameworks or reusable code that can be applied across the application or organization. As with any code or configuration change, all fixes should be thoroughly tested in non-operational environments.
