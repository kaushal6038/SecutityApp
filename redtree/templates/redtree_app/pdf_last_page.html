CONCLUSIONS

This vulnerability assessment is an analysis of the security posture as it was seen during the dates in which testing was conducted. Any future changes to the application or the infrastructure on which it resides may change the security posture from its current state. Additionally, as time passes, new types of attacks may arise which were previously unknown to the security industry. Because of this, Virtue Security recommends that all applications undergo recurring security testing within a reasonable timeframe.

Virtue Security also recommends that all issues identified in this report are remediated. Virtue Security makes a best effort to rank the severity of vulnerabilities; however, it is not possible for any assessment to guarantee that Low-risk issues cannot be used to significantly impact {{ client_info.client_name }}. All risk ratings in this report are a combination of industry standard ratings as well as considerations for unique business risk as identified by Virtue Security consultants. Special consideration should also be given to the fact that multiple low risk issues may compound into higher risk threats.

For most effective remediation, Virtue Security recommends developing processes or altering existing processes to include the fixes recommended in this report. Infrastructure related issues may be most effective when applied to baseline configurations which can be deployed across the organization. Application related fixes may be most effective with secure frameworks or reusable code that can be applied across the application or organization. As with any code or configuration change, all fixes should be thoroughly tested in non-operational environments.
